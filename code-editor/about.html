Language Extensions Token Coverage

Visual Design

Two themes, one codebase

Both themes are managed in separate CSS files and switched via a data-theme attribute on the <body> tag. Syntax colors match Visual Studio's palettes exactly — blue keywords, green comments, red-brown strings in light; cyan keywords, green comments, orange strings in dark.

Usage · Frontend

Using the editor UI

A tour of every toolbar control, panel, and keyboard shortcut. The interface is intentionally minimal — everything is one click or one keystroke away.

Toolbar controls

The toolbar at the top of the editor window is organized into five groups, separated by vertical dividers. From left to right:

  • Open — Loads a file from disk. Language is auto-detected from the extension.
  • Export — Downloads the current editor content as a file.
  • Language — Manually override the active syntax highlighter.
  • Theme — Switch between Light and Dark instantly.
  • Go to Line — Opens a validated line-jump dialog.
  • Symbols — Toggles the symbol navigator sidebar.
  • Diff — Toggles the git-style diff panel.
  • Completion API — Set and test your REST endpoint URL.

Panels & status bar

Three regions surround the main editing surface:

  • Symbol sidebar (left) — Lists every navigable symbol in the current document, grouped by kind. Click to jump.
  • Diff panel (right) — Shows added/removed/unchanged lines with line numbers and a +N -M summary.
  • Status bar (bottom) — Displays the current filename, active language, and caret line/column in real time.

The completion popup appears automatically when you type a trigger character (. or :) or press Ctrl+Space. Use ↑/↓ to navigate, Enter to accept, and Esc to dismiss.

Step-by-step: your first edit session

  1. Open or paste code

    Click Open in the toolbar (Ctrl+O) and pick a .vb or .r file. The language is detected automatically. Alternatively, just start typing — the editor works with an empty buffer too.

  2. Browse the document structure

    Click Symbols (Ctrl+B) to open the sidebar. Every function, class, module, property, and namespace is listed. Click any entry to jump there instantly.

  3. Fold code blocks

    Hover over the gutter on the left side of the editor. Clickable triangle markers appear next to foldable regions. Click to collapse a block — its content is hidden and a ... indicator is shown.

  4. Review your changes

    Click Diff (Ctrl+D) to open the diff panel. Lines you added appear in green with a + prefix; deleted lines appear in red with a -. The header shows the total counts.

  5. Get completions

    Type a . or :, or press Ctrl+Space. If a completion endpoint is configured and reachable, its items appear in a popup. Otherwise, the built-in fallback list is used.

  6. Export your work

    Click Export (Ctrl+S) to download the edited content. The original filename is preserved; for new documents the editor uses untitled.<ext>.

Usage · Backend

Configuring the completion API

The editor's intelligent completion feature is powered by a REST endpoint you control. Implement a single HTTP handler that accepts the document context and returns a list of completion items.

1. Endpoint contract

The editor sends an HTTP POST request with a JSON body describing the current editing context. The response must be a JSON object with an items array.

POST /api/completion Request Body 
{
  "language": "r",
  "text": "<full document text>",
  "line": 5,
  "column": 12,
  "trigger": "."
}
200 application/json Response Body 
{
  "items": [
    {
      "label": "mean",
      "kind": "function",
      "detail": "Arithmetic Mean",
      "documentation": "Computes the arithmetic mean of a numeric vector.",
      "insertText": "mean()"
    },
    {
      "label": "median",
      "kind": "function",
      "insertText": "median()"
    }
  ]
}

2. Request fields

  • language string The active language identifier: vbnet, r, json, xml, markdown, yaml, or plain.
  • text string The full document text at the moment the completion was requested. Use this for context-aware suggestions.
  • line number Zero-based line index of the caret position.
  • column number Zero-based column index of the caret position (UTF-16 code units).
  • trigger string? Optional. The character that triggered the request — typically . or :. Omitted when triggered by Ctrl+Space.

3. Response fields

  • items array Array of completion items (see below). May be empty.
  • items[].label string The text shown in the popup list. Required.
  • items[].kind string A category label such as function, variable, keyword, snippet, class, or property. Used for the icon.
  • items[].detail string? Optional short description shown next to the label (e.g. a type signature).
  • items[].documentation string? Optional longer documentation shown when the item is highlighted.
  • items[].insertText string? Optional text to insert when the item is accepted. If omitted, label is inserted.

4. Minimal backend example (Node.js)

This zero-dependency Node.js server implements the contract above. Save it as server.js and run with node server.js.

server.js 
const http = require('http');

http.createServer((req, res) => {
  if (req.method !== 'POST') {
    res.writeHead(405);
    return res.end();
  }

  let body = '';
  req.on('data', c => body += c);
  req.on('end', () => {
    const { language } = JSON.parse(body || '{}');

    const items = language === 'r'
      ? [
          { label: 'mean',   kind: 'function', insertText: 'mean()' },
          { label: 'median', kind: 'function', insertText: 'median()' },
          { label: 'sd',     kind: 'function', insertText: 'sd()' }
        ]
      : [
          { label: 'If',    kind: 'keyword' },
          { label: 'While', kind: 'keyword' }
        ];

    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ items }));
  });
}).listen(9000);

console.log('Completion server running on http://localhost:9000');

5. Connecting the editor to your backend

  1. Start your backend server

    Run the Node.js example above (or your own implementation in any language). Confirm it responds to a POST request with a JSON body.

  2. Enter the endpoint URL

    In the editor toolbar, find the Completion API group. Type the full URL into the text field — for example, http://localhost:9000/api/completion.

  3. Test the connection

    Click the Test button next to the URL field. The editor sends a probe request and shows the result: either OK: N items returned or an error message.

  4. Trigger completions while editing

    Type a . or :, or press Ctrl+Space. The editor POSTs the current document context to your endpoint and renders the returned items in a popup.

⚠️
CORS note: If your backend runs on a different origin than the editor page, it must return the Access-Control-Allow-Origin header. For development, set it to *. For production, restrict it to your editor's origin. The editor also sends an OPTIONS preflight request — your server should respond with status 204 and the appropriate Access-Control-Allow-* headers.
✓
Fallback behavior: If the endpoint is unreachable, returns an error, or takes too long, the editor silently falls back to a built-in suggestion list (VB.NET keywords or R built-in functions). You will never be left without completions.
Help

Frequently asked questions

Does the editor require any npm packages or external libraries?

No. The entire editor is built with plain HTML and TypeScript. The TypeScript source compiles to a single editor.bundle.js file using the outFile compiler option — no bundler, no runtime dependencies, no node_modules.

How do I rebuild the JavaScript bundle after editing the TypeScript source?

Install TypeScript 5.x globally (npm install -g typescript@5.6.3), then run tsc inside the project folder. The output is written to dist/editor.bundle.js. TypeScript 7+ removed the outFile option, so 5.x is required.

Can I add support for another programming language?

Yes. Create a new class that implements the ILanguageHighlighter interface (with tokenizeLine and initialState methods), register it in HighlighterRegistry.registerDefaults(), and add symbol extraction logic to SymbolNavigator. Recompile with tsc.

How is the diff computed?

The diff viewer uses a classic dynamic-programming Longest Common Subsequence (LCS) algorithm at the line level. It compares the originally loaded text against the current buffer, then backtracks through the DP table to produce a sequence of equal, added, and removed line entries.

What happens if my completion endpoint is slow or down?

The editor uses the browser's fetch API with a timeout. If the request fails or times out, it falls back to a built-in suggestion list appropriate for the current language. The user is never blocked — they can keep typing and the popup simply won't appear.

Are the themes customizable?

Completely. Every color is a CSS custom property defined in css/theme-light.css and css/theme-dark.css. To change a color, edit the variable value. To add a third theme, copy one of the files, change the selector to body[data-theme="mytheme"], adjust the variables, and add a new option to the theme dropdown in index.html.

Does the editor work offline?

Yes. Once the page is loaded, all editing, syntax highlighting, folding, symbol navigation, diffing, and go-to-line features work entirely offline. Only the REST completion feature requires network access — and it degrades gracefully to the built-in fallback when offline.
